An explanation for the schemas defined in userSchemaModel.js

The primary schema being referenced by MongoDB is as follow:
const GardenData = new Schema({
    users:[MemberSnapshotSchema],
    boxes:[BoxSchema]
})
Where users[] is a list of MemberSnapshotSchema objects and boxes[] is the same for BoxSchema.


Information on 'MemberSnapshotSchema'
-------------------------------
This schema holds the attributes for personal_info, member_info, and time_attr.
These categories are defined beneath the description for 'MemberSnapshotSchema'.
-------------------------------
id                  numerical iterator for unique id
                    PENDING_APPROVAL = 1;
                    ACTIVE = 2;
                    INACTIVE = 3;
updated_by          FOR AUDIT, string, person who changed box ownership
notes               String for now, maybe a list of strings later
box_ID              str, points to the ID of the box object for which they are either in line or of which they are a partial owner or NULL if none
